## 2. Verified Current Model

Verifiziert lesend gegen den echten Code (ohne Änderung). Die CLI ist reine
Orchestrierung der bestehenden lokalen Pipeline
(`HistoricalFileSource → MidBreakoutStrategy[/V1] → RiskEngine →
BacktestRunner → summarize_backtest_result → summary_to_markdown → Datei`);
keine Netzwerk-/Exchange-/Live-Pfade.

### CLI Contract

* Einstieg `main(argv: list[str] | None = None) -> int`.
* Argparse-Parser mit Argument-Gruppen (Data input, Strategy selection,
  Strategy parameters, Strategy v1 parameters, Risk / Backtest, Cost model,
  Output / Reporting).
* `--csv` und `--output` sind Pflicht; `--strategy` Default `v0`.

### Exit-Code Contract

* `_EXIT_OK = 0` — erfolgreicher Lauf, Report geschrieben.
* `_EXIT_USAGE = 2` — Nutzungs-/Validierungsfehler (argparse-Fehler,
  `_resolve_strategy_args`, `_validate_ranges`).
* `_EXIT_RUNTIME = 1` — Laufzeit-/Dateifehler (CSV fehlt, Output existiert ohne
  `--overwrite`, Auswertung wirft `ValueError`/`TypeError`).

### Validation Contract

* `_resolve_strategy_args`: v1-only-Parameter (`--breakout-threshold-pct`,
  `--cooldown-bars`, `--max-signals-per-day`) bei `--strategy v0` hart
  abgelehnt (Exit 2); Sentinel-`None` → strategieabhängige Defaults (v0/v1).
* `_validate_ranges` (nach Sentinel-Auflösung, fail-safe) lehnt u. a. ab:
  `--initial-equity <= 0`, `--lookback-bars <= 0`,
  `--stop-distance-pct` außerhalb `(0, 1)`, `--min-strength` außerhalb `[0, 1]`,
  v1: `--breakout-threshold-pct` außerhalb `[0, 0.1)`, `--cooldown-bars < 0`,
  `--max-signals-per-day <= 0` (oder weglassen),
  `--risk-per-trade-pct` außerhalb `(0, 1]`, `--max-position-size <= 0`,
  `--max-total-exposure <= 0`, `--max-daily-drawdown <= 0`, `--max-gaps < 0`,
  `--fee-rate/--spread/--slippage < 0`. Jeweils Exit 2.
* Die Strategie-Konstruktoren bleiben zusätzlich autoritativ (fail-safe).

### Bool Parsing Contract

* `_parse_bool` akzeptiert (case-insensitive): `true`/`1`/`yes` → `True`,
  `false`/`0`/`no` → `False`.
* Jeder andere Wert → `argparse.ArgumentTypeError` → argparse meldet auf stderr
  → `main` liefert Exit 2.

### Output/Framing Contract

* Geschriebener Inhalt = `f"{_DISCLAIMER}\n\n{markdown}\n"`:
  * Zeile 1: statischer Disclaimer-Kommentar
    (`<!-- Generated by Liquent local CLI. Research/backtesting only; not
    investment advice. -->`),
  * Zeile 2: Leerzeile,
  * danach der deterministische Markdown-Report,
  * abschließender Newline.
* Output ist deterministisch (keine Wall-Clock-Zeit, kein Zufall) — gleicher
  Input ergibt byte-identischen Report.

### Help Contract

* `--help` läuft erfolgreich und listet die Kern-Flags, Argument-Gruppen,
  den v0-Default und die v1-only-Parameter sowie die Cost-Model-Flags; enthält
  keine wertenden Begriffe.

### Error-Message Contract

* Erwartbare Fehler werden auf **stderr** im Format `error: …` gemeldet,
  **ohne** Traceback (argparse-Fehler werden von argparse selbst gemeldet).

### Terminal Output Contract

* Bei Erfolg auf stdout: `report written: <pfad>`, eine `strategy: …`-Zeile und
  eine `cost_model: …`-Zeile (rein informativ; der Markdown-Report bleibt davon
  unberührt).

### File Output Contract

* Fehlende Parent-Verzeichnisse des `--output`-Pfads werden via
  `mkdir(parents=True, exist_ok=True)` angelegt.
* Existiert die Output-Datei und `--overwrite` ist nicht gesetzt → Exit 1, Datei
  bleibt unverändert; mit `--overwrite` wird sie ersetzt.

### Descriptive-only / Safety Invariants

* Nur lokale Dateien; keine Netzwerk-Calls, keine Exchange-Anbindung, keine
  Zugangsdaten, keine Live-/Paper-Ausführung.
* Output trifft keine Aussage über zukünftige Ergebnisse; keine Bewertung,
  kein Ranking, keine Empfehlung.
* Ausdrücklich: keine neuen CLI-Flags, keine neuen Exit-Codes, keine neuen
  Output-Formate, kein Paper-Trading, kein Live-Trading, keine
  API-/Exchange-Anbindung; Test-Outputs nur über `tmp_path`.

## 3. Edge-Case Table

| Eingang | Ergebnis (aktuell) |
|---|---|
| `--initial-equity <= 0` | Exit 2 (Validierung) |
| `--lookback-bars <= 0` | Exit 2 (Validierung) |
| `--stop-distance-pct` `<= 0` oder `>= 1` | Exit 2 (Validierung) |
| `--max-position-size <= 0` | Exit 2 (Validierung) |
| `--max-total-exposure <= 0` | Exit 2 (Validierung) |
| `--max-daily-drawdown <= 0` | Exit 2 (Validierung) |
| `--allow-short` `true/false/1/0/yes/no` | akzeptiert (case-insensitive) |
| ungültiger `--allow-short`-Wert | Exit 2 (argparse) |
| fehlende `--csv` | Exit 1 |
| vorhandenes `--output` ohne `--overwrite` | Exit 1 |
| erfolgreicher Lauf | Exit 0 |
| fehlende `--output`-Parent-Verzeichnisse | werden angelegt |
| Report | beginnt mit Disclaimer, dann Leerzeile, endet mit Newline |

## 4. Regression Invariants

* Exit-Code-Trichotomie: Validierung/Usage → 2, Laufzeit/Datei → 1, Erfolg → 0.
* Output-Framing (Disclaimer + Leerzeile + Markdown + Newline) ist stabil und
  deterministisch.
* Fehler gehen nach stderr (`error: …`), ohne Traceback.
* Datei-Outputs erfolgen nur am angegebenen `--output`-Pfad; Parent-Dirs werden
  angelegt.
* CLI bleibt rein deskriptiv: keine Bewertung, kein Ranking, keine Empfehlung.

ausschließlich **bestehendes** Verhalten fest (keine neuen Features, keine
Änderung bestehender CLI-Tests), via `main([...])` mit `tmp_path` und den
vorhandenen Fixtures.

Adoptierte Lücken (echtes, bisher nicht explizit auf CLI-Ebene gesichertes
Verhalten):

* **validation** — `--initial-equity <= 0` → Exit 2.
* **validation** — `--lookback-bars <= 0` → Exit 2 (CLI-Ebene `_validate_ranges`).
* **validation** — `--stop-distance-pct` `<= 0` und `>= 1` → Exit 2 (CLI-Ebene).
* **validation** — `--max-position-size <= 0` → Exit 2.
* **validation** — `--max-total-exposure <= 0` → Exit 2.
* **validation** — `--max-daily-drawdown <= 0` → Exit 2.
* **bool** — `--allow-short` akzeptiert `true/false/1/0/yes/no` (case-insensitive).
* **bool** — ungültiger `--allow-short`-Wert → Exit 2.
* **exit codes** — Trichotomie in einem Test: Validierung → 2, fehlende CSV → 1,
  vorhandenes Output ohne `--overwrite` → 1, Erfolg → 0.
