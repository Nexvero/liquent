## Safety boundaries

* No live trading
* No paper trading
* No exchange connection
* No API keys
* No external data download by Liquent
* No uploaded CSV persistence
* No report files generated by the preview
* No profitability assessment
* No trading recommendation

## Recommended next steps

* Keep the quickstart and index updated as Visual Preview evolves.
* Keep CSV examples as code blocks, not committed CSV files.
* Only later and only if explicitly specified: controlled local runner integration.
* Keep all live, paper, exchange, and API functionality out of the Visual Preview.
